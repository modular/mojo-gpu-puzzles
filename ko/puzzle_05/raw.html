개요

벡터 a와 b를 브로드캐스트로 더해 2D 행렬 output에 저장하는 커널을 구현해 보세요.

참고: 스레드 수가 행렬의 위치 수보다 많습니다.

핵심 개념

이 퍼즐에서 배울 내용:

  • 1D 벡터를 각각 다른 차원 방향으로 브로드캐스트하기
  • 2D 스레드 인덱스로 브로드캐스트 연산 수행하기
  • 브로드캐스트 패턴에서 경계 조건 처리하기

핵심은 두 1D 벡터의 원소들을 브로드캐스트로 2D 출력 행렬에 매핑하는 방법을 이해하고, 스레드 경계를 올바르게 처리하는 것입니다.

  • 브로드캐스트: a의 각 원소가 b의 각 원소와 결합
  • 스레드 매핑: \(2 \times 2\) 출력에 \((3 \times 3)\) 스레드 그리드 사용
  • 벡터 접근: a와 b는 서로 다른 접근 패턴 사용
  • 경계 검사: 행렬 범위를 벗어나는 스레드를 가드로 처리

팁

  2. 가드 추가: if row < size and col < size
  3. 가드 내부: a와 b 값을 어떻게 브로드캐스트할지 생각해 보세요

솔루션

def broadcast_add(
    output: UnsafePointer[Scalar[dtype], MutAnyOrigin],
    a: UnsafePointer[Scalar[dtype], MutAnyOrigin],
    b: UnsafePointer[Scalar[dtype], MutAnyOrigin],
    size: Int,
):
    var row = thread_idx.y
    var col = thread_idx.x
    if row < size and col < size:
        output[row * size + col] = a[col] + b[row]

LayoutTensor 추상화 없이 GPU 브로드캐스트의 기본 개념을 보여주는 솔루션입니다:

  1. 스레드에서 행렬로 매핑

    • thread_idx.y로 행, thread_idx.x로 열에 접근
    • 2D 스레드 그리드를 출력 행렬 원소에 직접 매핑
    • 3×3 그리드의 초과 스레드를 2×2 출력에 맞게 처리

  2. 브로드캐스트 작동 방식

    • 벡터 a는 수평 방향으로 브로드캐스트: 각 행에서 동일한 a[col] 사용
    • 벡터 b는 수직 방향으로 브로드캐스트: 각 열에서 동일한 b[row] 사용
    • 두 벡터를 더해 출력 생성
    [ a0 a1 ]  +  [ b0 ]  =  [ a0+b0  a1+b0 ]
              [ b1 ]     [ a0+b1  a1+b1 ]

  3. 경계 검사

    • 단일 가드 조건 row < size and col < size로 두 차원 모두 처리
    • 입력 벡터와 출력 행렬의 범위 초과 접근 방지
    • 3×3 스레드 그리드가 2×2 데이터보다 크므로 반드시 필요

LayoutTensor 버전과 비교해서 동일한 기본 개념을 유지하면서 추상화가 브로드캐스트 연산을 얼마나 단순하게 만드는지 확인해 보세요.
